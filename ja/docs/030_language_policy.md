# 言語ポリシー（英語を正本 / 日本語はコピー）

このリポジトリでは、意図的に **2つの言語ツリー**を併存させます。

- 英語: `docs/` と `agent/`（運用の正本 / XID 管理対象）
- 日本語: `ja/docs/` と `ja/agent/`（人間向けの日本語コピー/メモ）

## 英日間を XID で相互参照しない理由

XID は、改名・移動・分割・統合があっても参照を安定させるための主キーです。これを安定して運用するには、**XID 管理対象の正本が1つ**であることが重要です。

両言語を同一の XID インデックスで管理すると、次の問題が避けられません。

- 翻訳により **XID が重複**しやすい
- `rewrite/check` の対象が曖昧になる（どの言語へ解決すべきか）
- 運用コストが増える割に得られる効果が小さい（日本語側は主に人間向け）

そのため、次の方針を採用します。

- 管理リンク（`#xid-...`）は **英語側（`docs/` / `agent/`）で完結**させる
- `ja/` は **XID インデックスの対象外**とし、`xref rewrite/check` の管理対象にしない

## 英日間の導線が必要な場合

言語間の行き来が必要な場合は、XID 管理リンクではなく、シンプルな“案内リンク”を置きます。

- 英語側: `- Japanese: ../ja/docs/<path>.md`
- 日本語側: `- English: ../../docs/<path>.md`

これらは XRefKit の自動書き換え対象ではないため、最小限にし、パスが安定するように運用します。

## この設計における「正本」

- AI が読む・参照する正本は `docs/`（XID 管理対象）
- 人間が検証できる原本は `sources/`
- `ja/` は人間向けのコピーで、参照グラフ（XID）とは分離する
