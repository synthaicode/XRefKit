# XRefKit リポジトリ説明動画シナリオ

この動画は、現在の XRefKit リポジトリが何を管理しているかを、初見でも追える順番で短く説明する。

## ゴール

- 視聴者が、XRefKit を単なる文書置き場やリンク維持ツールではなく、AI 作業の運用基盤として理解する。
- 視聴者が、`docs / flows / capabilities / skills / knowledge / work / sources` の役割分担を理解する。
- 視聴者が、Skill runtime envelope、context direction guard、handoff 記録、人間判断境界がこのリポジトリの現在の中核であると理解する。

## 対象者

- このリポジトリを初めて見る人
- AI を業務へ入れたいが、どう構造化すべきかを知りたい人
- 他の AI や人へ XRefKit を説明しなければならない人

## 構造

1. このリポジトリは何か
2. なぜ文書置き場では足りないか
3. どの層で AI を制御するか
4. Skill 実行をどう囲うか
5. 外部入力で方針が崩れないようにどう守るか
6. 継続作業と複数 AI をどう引き継ぐか
7. 人間はどこで判断するか
8. まとめ
