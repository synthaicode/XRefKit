# 既存ドキュメント取り込み（人間向けの見え方）

このリポジトリの `docs/` が分割されているのは、このシステム自体の説明と運用（XID/リンク整合）を保つためです。

**取り込みたい既存ドキュメント（PDF/Excel/Web 等）を、事前に分割して用意する必要はありません。**

## 取り込みの基本（元データは分割しない）

1. 元データをそのまま `sources/` に置く（PDF/Excel/HTML など）
2. AI が必要な箇所だけを参照し、`docs/` に “1ページ=1断片” の Markdown として抽出結果を作る
3. 断片 Markdown の末尾に、必ず出典（`sources/` のパス＋位置情報）を残す

## なぜ `docs/` 側は分割するのか

原本が巨大でも、AI が毎回全体を読む必要がないようにするためです。

- `sources/`: 人間が確認するための原本保管庫
- `docs/`: AI が参照する知識の断片（XID付き）
