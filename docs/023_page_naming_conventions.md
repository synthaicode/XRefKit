# Page Naming Conventions

This page defines the naming conventions used for major page types in `docs/`.

## General Rule

Use Title Case for page titles.

Keep the page-type word explicit when the page belongs to a known family such as:

- `Overview`
- `Index`
- `Structure`
- `Workflow`
- `Policy`
- `Protocol`
- `Guide`
- `Operating Model`
- `Design`
- `Mechanism`
- `Register`
- `Schema`
- `Boundaries`

## Qualifier Rule

When a qualifier is needed, prefer one of these patterns:

- `<Subject> <Page Type>`
- `<Subject> (<Qualifier>)`

Examples:

- `Repository Structure (Human View)`
- `Language Policy (English Canonical + Japanese Archive)`
- `Docs Index (Entry)`

## Family Consistency

- workflow pages should end with `Workflow`
- policy pages should end with `Policy` when practical
- usage pages should end with `Usage Guide`
- operating-model pages should end with `Operating Model`
- design pages should end with `Design`
- register pages should end with `Register`

## Scope Note

These conventions are for readability and consistency.
They do not require renaming every existing file path immediately.
Prefer updating page titles and index labels first.
